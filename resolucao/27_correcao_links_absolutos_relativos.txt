=== CORREÇÃO DE LINKS ABSOLUTOS PARA RELATIVOS ===
Data: 24/08/2025
Problema: Links quebrados após redirecionamento da raiz

== PROBLEMA IDENTIFICADO ==
Após implementar o redirecionamento da raiz (/) para /frontend/ no router.php, 
as páginas perderam as referências porque usavam caminhos absolutos que 
apontavam para a raiz do servidor em vez da pasta frontend.

Erros 404 observados:
- GET /login.html (deveria ser /frontend/login.html)
- GET /assets/css/style.css (deveria ser /frontend/assets/css/style.css)
- GET /assets/js/main.js (deveria ser /frontend/assets/js/main.js)

== ANÁLISE DO PROBLEMA ==
1. Links HTML usavam caminhos absolutos: href="/login.html"
2. Assets CSS/JS usavam caminhos absolutos: href="/assets/css/style.css"
3. Navegador procurava arquivos na raiz (localhost:8000/) em vez de /frontend/
4. Resultado: páginas não carregavam corretamente

== SOLUÇÃO IMPLEMENTADA ==

### 1. Correção de Links HTML
Alterados de caminhos absolutos para relativos:

**Páginas corrigidas:**
- frontend/index.html
- frontend/login.html
- frontend/cadastro.html
- frontend/perfil.html
- frontend/categoria.html
- frontend/noticia.html

**Exemplos de correções:**
- href="/login.html" → href="login.html"
- href="/cadastro.html" → href="cadastro.html"
- href="/perfil.html" → href="perfil.html"
- href="/favoritos.html" → href="favoritos.html"
- href="/newsletter.html" → href="newsletter.html"

### 2. Correção de Assets CSS/JS
Alterados de caminhos absolutos para relativos:

**Assets corrigidos:**
- href="/assets/css/style.css" → href="assets/css/style.css"
- src="/assets/js/main.js" → src="assets/js/main.js"

**Páginas com assets corrigidos:**
- frontend/login.html
- frontend/cadastro.html
- frontend/categoria.html
- frontend/noticia.html

### 3. Verificação do Painel Admin
As páginas administrativas (admin/*.html) já usavam caminhos relativos corretos:
- href="../perfil.html" (correto)
- src="../assets/js/main.js" (correto)
- href="assets/css/admin.css" (correto)
